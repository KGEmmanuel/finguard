# Security Policy

## Reporting a vulnerability

Please report suspected security issues **privately**. Two channels,
either is fine:

1. **GitHub private vulnerability reporting** — open a report from the
   Security tab of [KGEmmanuel/vigil-mesh](https://github.com/KGEmmanuel/vigil-mesh/security/advisories/new).
   Preferred: keeps the discussion and fix in one place.
2. **Email** — [security@cognitagrc.io](mailto:security@cognitagrc.io).
   A PGP key is available on request.

Include:
- A description of the issue and its impact
- Steps to reproduce (PoC if possible)
- Affected versions / commits
- Your name for credit (optional)

We aim to acknowledge reports within 3 business days and to ship a fix or mitigation within 30 days for high-severity issues. Please do not disclose publicly until we have released a fix.

## Supported versions

The `main` branch and the latest tagged release receive security fixes.

## Scope

In scope: this repository ([KGEmmanuel/vigil-mesh](https://github.com/KGEmmanuel/vigil-mesh)). Out of scope: the hosted Cognita GRC platform (report those to the same address; we route internally).
